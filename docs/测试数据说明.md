# 测试数据说明（端到端演示走查）

本文档列出演示环境全部种子脚本的执行顺序、每份数据的用途与对应演示路径。

## 一、种子脚本执行顺序

```bash
# 1. 租户 + 用户（T001 / admin 等演示账号）
python scripts/seed_tenants.py

# 2. 38 份预置制度文档（复制 + 元数据 + 向量索引）
python scripts/seed_regulations.py

# 3. 3 个内置场景包（G01 / G11 / EAST_JJ）
python scripts/seed_report_packs.py

# 4. 22 条术语词典
python scripts/seed_terms.py

# 5. 演示补充数据（本文件主角，依赖前四个已跑）
python scripts/seed_demo_extra.py
```

全部脚本幂等，可重复执行；`seed_demo_extra.py` 已连跑两次验证幂等。

## 二、seed_demo_extra.py 产出清单

### ① 制度新旧两版演示文档对（制度预演 / 版本 diff 演示）

| 文件 | 位置 | 用途 |
| --- | --- | --- |
| 源文件（demo_ 前缀） | `data/tenants/T001/regulations/demo_1104_G11_五级分类_2024版.txt` | 脚本文本源，随仓库分发 |
| 源文件（demo_ 前缀） | `data/tenants/T001/regulations/demo_1104_G11_五级分类_2025修订版.txt` | 同上 |
| 入库副本 | `data/tenants/T001/regulations/{doc_id}_1104_G11_五级分类_2024版.txt` | 向量库已索引（doc_type=1104） |
| 入库副本 | `data/tenants/T001/regulations/{doc_id}_1104_G11_五级分类_2025修订版.txt` | 同上 |

内容基于 `02_制度文档向量库/01_1104非现场监管报表/G11_资产质量五级分类.txt` 改造，
2025 修订版相对 2024 版有 3 处口径差异，供 diff 演示：

1. **逾期整笔认定临界点 90 → 91 天**（2024 版逾期 90 天即按整笔本金填报不良，
   2025 版放宽为 91 天及以上）；
2. **新增「关注类细分条款」**（关注一 / 关注二）；
3. **删除「核销贷款账销案存次月排除」条款**（改为提示核销贷款不再在 G11 重复列示）。

演示路径：制度预演 / 文档版本对比接口，选这两份文档做新旧对比。

### ② 预造 completed 的 G11 演示任务

- 任务 ID：`TASK_DEMO_G11_001`（租户 T001，1104 G11，场景包 G11，目标表
  `rpt_g11_five_classify`，6 个阶段全绿，created_by=demo）。
- 方案库案例：任务完成时自动沉淀 `solution_cases`（pack=G11），
  供「相似方案推荐」接口演示。
- 注意（离线 Mock 限制）：Mock AI 适配器只生成 G01 形态固定 SQL，缺
  `five_classify` 列，无法通过 G11 场景包的 `SUM_BY(five_classify, ...)`
  分组勾稽，直连跑会被质量门禁回退 3 次判 failed。脚本的处理是：保留真实跑出的
  regulation_parser 产出，codegen 替换为按 G11 target_schema 手工编写的 SQL
  （脚本内 `DEMO_G11_SQL`，含 90 天分界 overdue_principal 列），从
  quality_gate 断点续跑——后续 Agent（质量门禁 / 测试验证 / 数字孪生 /
  投产交付）全部真实执行、真实通过，产物可复核。接入真实 AI Key 后无需此兜底。

演示路径：任务详情页（阶段明细）、方案库推荐、台账关联、接口文件导出。

### ③ 当月报送台账

- 调 `ledger_service.generate_ledger(T001, 当月)`：按 3 个 active 场景包生成
  当月台账条目（截止期 = 次月 5 日 23:59:59），幂等。
- G11 条目自动绑定 `TASK_DEMO_G11_001`（状态升级为 in_progress），
  供「台账关联任务」演示。
- 若范围 A 台账服务未就绪，脚本 try/except 打印跳过提示，不影响其他步骤。

### ④ 新旧回归演示 SQL

| 文件 | 说明 |
| --- | --- |
| `data/tasks/demo_regression/old.sql` | 2024 版口径：逾期 ≥90 天按整笔本金填报不良 |
| `data/tasks/demo_regression/new.sql` | 2025 版口径：临界点 91 天 + 关注类细分标签 |

两版 SQL 均可通过只读护栏并真实执行。配合 ⑤ 的临界数据，差异可解释：
不良总额 old=2,425,000 vs new=2,000,000，差 -425,000，全部来自 90 天那笔
借据 C901（旧口径整笔 500,000 → 新口径按逾期部分估算 75,000）。

演示路径：数字孪生「新旧逻辑回归」接口（report_pack_id=G11，粘贴两段 SQL）。

### ⑤ 逾期临界演示借据（demo_biz.db）

幂等 `INSERT OR IGNORE` 两笔借据，让 90→91 口径变化产生非零回归差异：

| contract_no | overdue_days | principal_balance | 说明 |
| --- | --- | --- | --- |
| C901 | 90 | 500,000 | 旧口径整笔 / 新口径按部分，差异来源 |
| C902 | 91 | 650,000 | 新旧口径均整笔，对照组 |
