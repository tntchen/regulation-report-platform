# 开发环境一键脚本说明（scripts/setup_dev.sh）

> 适用对象：首次拉取仓库、或需要重建本地开发环境的同事。
> 一句话：`bash scripts/setup_dev.sh` 从干净仓库到可启动服务，全程无需手工干预。

## 1. 脚本做什么

脚本按固定顺序执行 5 个阶段，任一阶段失败即中止（`set -euo pipefail`），不会留下"半装好"的环境：

| 阶段 | 动作 | 失败时的表现 |
|---|---|---|
| 1. 选解释器 | 按 `python3.12 → 3.11 → 3.10 → 3.13 → python3` 顺序探测，只接受 **3.10~3.13** | 报错退出并提示 `brew install python@3.12` |
| 2. 建虚拟环境 | 在项目根目录创建 `.venv`（已存在则跳过），升级 pip 后安装 `requirements.txt` | pip 报错原样输出 |
| 3. 补 `.env` | `.env` 不存在且 `.env.example` 存在时复制一份 | 静默跳过（两者都在/都不在都不报错） |
| 4. 灌演示数据 | 按序执行 5 个种子脚本（全部幂等） | 种子脚本报错即中止 |
| 5. 启动服务 | 仅加 `--serve` 时执行，`exec uvicorn` 前台运行 | Ctrl+C 停止 |

## 2. 为什么限制 Python 3.10~3.13

依赖链里的 **torch / sentence-transformers** 目前没有 Python 3.14 的预编译 wheel，在 3.14 上 pip 会卡在源码编译或直接失败。脚本在第一步就拒绝 3.14，把失败前置到装依赖之前，避免浪费时间。这也是探测顺序把 `python3.12` 放最前、把裸 `python3` 放最后兜底的原因——macOS 上裸 `python3` 经常是 Homebrew 最新版（当前为 3.14）。

代码本身兼容 3.10+，限制只来自 ML 依赖的 wheel 可用性。

## 3. 命令行参数

| 参数 | 行为 | 典型场景 |
|---|---|---|
| 无参数 | 装环境 + 灌数据，不启动服务 | 首次搭建 |
| `--serve` | 全流程结束后前台启动 `uvicorn backend.main:app --port 8080 --reload` | 搭建完直接联调 |
| `--skip-seed` | 只装环境，不跑种子脚本 | 已有数据、只想重建 venv |
| `--tfidf` | 安装时剔除 `sentence-transformers` / `scikit-learn`，并导出 `EMBEDDING_PROVIDER=tfidf` | 装不了 torch 的机器（旧系统、无网环境）；语义检索降级为关键词模式，其余功能不变 |
| `-h` / `--help` | 打印脚本头部注释 | — |

## 4. 种子脚本执行顺序与幂等性

```
seed_tenants        租户 + 账号（admin / zhangsan）
seed_regulations    监管制度库
seed_report_packs   场景包 + 预置演示任务 TASK_DEMO_G11_001
seed_terms          术语库
seed_demo_extra     P2 演示数据（报送台账、接口文件、回归 SQL、临界借据）
```

顺序有依赖：租户是所有数据的归属，场景包引用制度库条目，demo_extra 绑定预置任务。**全部幂等**——重复执行只跳过已存在的数据，因此脚本可以反复重跑（例如 pip 中断后直接再跑一遍，已完成的种子会自动跳过）。

## 5. 已知边界与排错

- **pip 哈希校验报错**（`THESE PACKAGES DO NOT MATCH THE HASHES`）：网络抖动导致的缓存损坏。执行 `.venv/bin/pip cache purge` 后重跑脚本即可。
- **torch 下载慢**：首次安装约 200MB+。无网或弱网环境直接用 `--tfidf`。
- **BGE 模型首次运行需联网**（约 100MB，HuggingFace 缓存后离线可用）；无网时同样走 `--tfidf`。
- **换 Python 版本重建**：删除 `.venv` 目录后重跑脚本。
- **.venv 不入库**：已加入 `.gitignore`，不要手工提交。
